## 1. 定义与解释

在本协议中，下列词语具有以下含义：

### 1.1 核心定义

* **本仓库/本软件**：指由[仓库维护者]托管于 GitHub/GitLab/Gitee 等平台的源代码仓库，包括其所有源代码、资源文件、构建产物（Jar/Zip等）、文档及相关脚本。
* **维护者**：指 饩雨/Mai_xiyu/God_xiyu 及其授权的协作开发者。
* **使用者**：指以任何方式下载、复制、安装、运行、修改或分发本仓库内容的主体。

### 1.2 作品形态定义（关键区分）

* **衍生作品 (Derivative Works)**：指基于本仓库源代码进行修改、翻译、重构、移植，或直接复制本仓库代码片段所形成的新作品。包括但不限于：Fork后的仓库、修改版模组、包含本仓库代码的整合包核心等。
* **附属/联动作品 (Dependent Works)**：指**不包含**本仓库源代码，仅通过 API、事件系统（Event System）、反射（Reflection）或数据包（Datapack）形式与本仓库进行交互、调用或扩展功能的独立作品。例如：附属模组、联动插件、依赖本模组作为前置的模组。

## 2. 许可分层策略

### 2.1 针对衍生作品的许可（传染性闭源）

任何被认定为**衍生作品**的项目，必须严格遵循**本协议的所有条款**（即本协议第3、4、5条限制）。

* **禁止协议变更**：衍生作品不得更换为更宽松的开源协议（如 MIT, Apache 2.0 等）。
* **继承性**：衍生作品发布时，必须在显著位置包含本协议全文。

### 2.2 针对附属/联动作品的许可（开源与自主）

任何以本仓库为**前置依赖**、或作为**附属模组**发布的软件作品：

1. **开源要求**：必须采用 **GNU General Public License v3.0 (GNU GPL v3.0)** 或其后续版本进行开源。
2. **分发自主权**：在遵守 GPL v3.0 的前提下，附属作品的作者享有独立的分发权利（详见 3.2.1 例外条款）。

## 3. 严格使用限制

### 3.1 禁止商业使用 (Non-Commercial)

使用者不得将本仓库内容（含衍生作品）用于任何商业目的。
**禁止行为包括但不限于：**

1. **付费产品**：将代码嵌入收费软件、服务器核心、登入验证器或硬件设备中。
2. **盈利性服务**：将本软件作为收费服务器（Pay-to-Win或会员制）的专属功能或卖点。
3. **直接/间接获利**：出售本软件的下载链接、通过本软件进行广告引流、或将其作为众筹/赞助的回报物。
4. **倒卖与授权**：禁止出售、出租或转授权本软件的任何部分。

### 3.2 禁止发布至特定平台 (Anti-Platform)

为防止未经授权的整合与滥用，**严禁**将本仓库内容（含衍生作品及构建产物）发布、上传、分发至**网易公司（NetEase）及其关联方**运营或控制的任何平台。
**违禁范围明确包括：**

1. 网易我的世界中国版（包含租赁服、网络游戏大厅、组件中心）。
2. 网易云信、网易游戏开发者平台。
3. 网易投资或控制的第三方模组托管平台（若有）。

#### 3.2.1 附属/联动模组的豁免（新增条款）

**本限制（第 3.2 条）不适用于“附属/联动作品”。**
由第三方开发的附属或联动模组，其发布平台由该作品的作者自行决定。**附属模组作者有权将其开发的附属模组发布至包括网易在内的任何平台**，无需征得本仓库维护者同意。

### 3.3 完整性与归属保留

1. **禁止篡改元数据**：不得删除或修改代码中的 `mcmod.info`、`fabric.mod.json`、`mods.toml` 等文件中的作者信息（Authors）、协议声明（License）及链接。
2. **Fork 限制**：Fork 后的仓库必须在 README 文件的顶部显著位置保留指向原仓库的链接及本版权声明。
